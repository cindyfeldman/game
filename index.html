Song Quiz

by: Cindy Feldman

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I wanted a program that would combine both games and music. Because I didn't want to make another boring game, I decided to make a quiz. This way, I get to enjoy listening to the songs while people play Song Quiz.

2. What I Learned

I learned that not a lot of people were familiar with the songs. I also learned that changing the color of the frame depending on the user's answer would make it more obvious if their guess was correct or not. I proceeded to add a green background for correct answers and red for incorrect answers.

3. Architecture of My Game

The Game Panel class takes care of the different states of the game. It controls what happens when a key is pressed and what each state does. The Quiz Frame draws the game state of the game, the JFrame. This contains the four questions with their songs. When the last question is answered, the user will press the 'get score' button to get to the end state. This state tells the user their score and then allows them to exit the game when they press 'exit'.

4. Technical Implementation

I am most proud of the end state of the game. Even though it just tells the game user their score and allows them to exit the game, it took a lot of trial and error to accomplish. When the last question is answered and the button is pressed, The 'get score' button will be enabled. When this button is pressed, four song name option button are disposed of. A label with the user's score appears and the 'exit' button is enabled. Pressing the exit button simply disposes the quiz JFrame. My goal was to create a quiz game that would calculate the user's score at the end. To accomplish this, I first worked on the quiz frame and dealt with each question and having the song play at the right time. After, I worked on switching the states from Menu to Game. Lastly, I worked on the end State and created buttons that would take the user to the end state.

5. Most Challenging Part

I struggled with changing the states of the game the most. At first, I couldn't figure out how to get from the menu state to the game state because the game state was a JFrame. I tried a lot of different methods before figuring out that I should call the quiz frame in the action performed method, when the button is pressed.

6. Future Fixes and Updates

I would add more questions. I would also have the score depending on how quick the user guesses the song.
